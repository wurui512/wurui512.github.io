js相关总结

Veröffentlicht am 2018-01-24 | in javaScript

平时的一些总结

事件代理

事件代理就是利用事件冒泡，就是用一个事件处理程序来管理一类事件。因为在js中添加在页面中的事件处理程序数量将直接影响页面的整体性能。如果用事件委托，那么与dom的操作只需要交互一次，就大大减少了与dom的交互次数，提高性能。

JS中的this是怎么工作的

this表示函数作用域的上下文，指向函数运行时所在的对象，即调用它的对象。

JS原型继承的原理

Js的继承主要依靠__proto__，每个对象都有__proto__这个属性，实例的__proto__指向其构造函数的原型对象。当js引擎查找对象属性时先在自身上找，没有的话在其原型链上找，即在构造函数的原型对象上找。每个函数对象都由prototype属性，这个属性指向函数的原型对象。

commonJS和AMD/CMD

js模块化的三个规范：CommonJS，AMD，CMD。
CommonJS定义的模块分为：require、export、module. 但require加载模块是同步的，必须等模块加载出来才能执行后面的操作，这在服务器端是完全可行的，因为所有模块都存在本地硬盘，可以同步加载完成，加载时间是硬盘读取时间。但在浏览器端，因为模块都存在服务器，加载时间取决于网速，可能要等很长时间，浏览器会处于“假死”状态。
于是就诞生了AMD（异步模块定义），采用异步加载模块，模块加载不影响后面语句的运行，所有依赖这个模块的语句都放在回调函数中。AMD也采用require()，但不同于CommonJS，它采用require([module], callback);相比于CommonJS，适合浏览器端。

null、undefined、undeclared的区别，怎么检测

Undeclared是js语法错误，表示变量没有声明，js无法找到对应的上下文。
Undefined表示一个变量没有被声明，或者被声明了没有被赋值。如果一个函数没有返回任何值则默认返回undefined。
Null是一个表示“没有值”的值。
null可以自动转为0，undefined转换成数值是NaN。
undefined不是一个有效的json，而null是。
undefined的typeof是undefined。null的typeof是object。
判断一个变量是否是undefined：typeof variable ===”undefined”。
判断一个变量是否是null：variable === null

什么是闭包，如何使用，为什么要使用？

闭包是指有权访问另一个函数作用域中的变量的函数。
创建闭包的方式是在一个函数内返回另一个函数。
闭包的作用：1、可以读取函数内部的变量。2、让变量的值始终保存在内存中，不会被自动清除。

JSONP和AJAX的区别联系

ajax的核心是通过xmlHttpRequest获取非本页内容
jsonp的核心是动态添加script标签调用服务器提供的js脚本（后缀.json)

常见的创建dom节点的方式

  • createElement() 创建一个元素节点 => 接收参数为string类型的nodename
  • createTextNode() 创建一个文本节点 => 接收参数为string类型的text内容
  • createAttribute() 创建一个属性节点 => 接收参数为string类型的属性名称
  • createComment() 创建一个注释节点 => 接收参数为string类型的注释文本
  • createDocumentFragment() 创建一个虚拟的节点对象 => 接收参数为string类型的nodename
