原型链

Veröffentlicht am 2017-10-22 | in javaScript

继承

ECMA实现继承主要是依靠原型链。原型链的主要思想是让一个引用类型继承另一个引用类型的属性和方法.
理解原型链之前必须先了解构造函数，原型，实例之间的关系。

构造函数、原型、实例：

//构造函数、原型、实例的关系
firend1.constructor == Person;
alert(friend1 instanceof Person);//true
alert(friend1 instanceof Object);//true
friend1.__proto__== Person.prototype; // __proto__隐式原型

原型链的问题

1、已经知道包含引用类型值的属性会被所有实例共享，所以要在构造函数中创建引用类型，而不能在原型中定义属性的原因。所以组合使用构造函数模式和原型模式是最常见方式。如下图。

但是，利用原型链来实现继承时，原型会变成另一个类型的实例，于是原先的实例属性就会变成现在的原型属性了。

SuperType构造函数定义了一个colors属性，则SuperType的所有实例都会共享这个属性，SubType继承了SuperType，则SubType.prototype就是SuperType的一个实例，所以它也有自己的colos属性，向instance1添加值其实就是改变SubType.prototype.colors的属性，所以SubType的所有实例都会共享这个属性。
2、创建子类型实例时不能向超类型的构造函数中传递参数。

原型链问题的解决方法

组合式继承

第一次调用SuperType构造函数时，SubType.prototype会得到两个属性：name和colors。当调用SubType构造函数时，又一次调用SuperType构造函数，这一次又在新对象上创建了实例属性name和colors,于是这两个属性就屏蔽了原型中的两个同名属性。
组合式继承避免了原型链和借用构造函数的缺陷，融合了它们的优点，称为JS中最常用的继承模式。使用instanceof()和isPrototypeOf()可以识别基于组合继承创建的对象。
但是组合式继承会调用两次超类型构造函数，于是就有了两组name和colors属性，针对这个问题有了寄生式组合继承这个解决方法。
